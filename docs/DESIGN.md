## 1. 设计原则

本设计规范严格遵循 `PRD.md` 中确定的产品定位与 `prototype/index.html` 中已验证的实现细节，禁止杜撰未在指定文件中出现的内容。

### 1.1 风格定位

- **整体风格**：水墨国风 + 2D 横板表现。
- **情感基调**：留白、沉静、可顿悟，避免过度装饰与强刺激。
- **视觉隐喻**：
  - 宣纸底纹 → 卷轴、弹窗、叙事框。
  - 墨色渐变 → 按钮悬停、水流、阴影。
  - 印章 → 通关、节俭、成就标记。
  - 卷轴边角 → 按钮、弹窗、关卡地图。

### 1.2 设计目标

1. **文化感**：通过色彩、字体、留白传递国风气质，而非堆砌传统元素。
2. **可读性**：解谜信息（金钱、状态、库存）在柔和背景上保持高对比。
3. **反馈即时**：编辑、放水、模拟、结算各阶段的状态切换在 3 秒内可被玩家感知。
4. **零惩罚体验**：失败状态视觉不刺眼，重试入口始终突出。

## 2. 色彩系统

色彩全部提取自 `prototype/index.html` 中的 CSS 变量，后续实现必须保持一致。

### 2.1 主色调

| 变量名 | 色值 | 用途 | 使用场景 |
|--------|------|------|----------|
| `--ink-black` | `#1a1a1a` | 主墨色 | 标题文字、主按钮背景、重要边框 |
| `--ink-dark` | `#2d2d2d` | 深灰墨 | 正文、二级按钮边框、弹窗边框 |
| `--ink-gray` | `#5a5a5a` | 中灰 | 次要说明文字、状态描述 |
| `--ink-light` | `#8a8a8a` | 浅灰 | 禁用态、占位文字 |
| `--paper` | `#f5f0e8` | 宣纸白 | 页面背景、弹窗底色、按钮底色 |
| `--paper-dark` | `#e8e0d4` | 陈年纸 | 进度条底色、金钱条底色、加载背景 |
| `--ochre` | `#c4a77d` | 赭石 | 卷轴轴杆、选中高亮、预览解锁状态 |
| `--green-pale` | `#a8c5b5` | 淡青绿 | 编辑中状态、节俭成功、已通关标记 |
| `--red-ink` | `#8b3a3a` | 朱砂 | 失败状态、删除区、印章、村庄受击 |
| `--water` | `#3a4a5a` | 水色 | 放水按钮、河流背景 |
| `--water-light` | `#5a7a8a` | 浅水色 | 河流渐变、水源标识 |
| `--shadow` | `rgba(0,0,0,0.15)` | 投影 | 按钮、弹窗、关卡节点阴影 |

### 2.2 色彩使用规则

1. **背景层级**：
   - 最底层：`--paper` + `--paper-dark` 纵向渐变。
   - 中景层：半透明 `--water-light` 河流区域。
   - 前景层：`--paper` 卡片、弹窗、工具栏。

2. **状态语义**：
   - 成功 / 安全：`--green-pale`。
   - 进行中 / 警告：`--ochre`。
   - 失败 / 危险：`--red-ink`。
   - 默认 / 可交互：`--ink-dark` 边框 + `--paper` 底色。

3. **禁用态**：
   - 透明度降至 `0.5`。
   - 光标变为 `not-allowed`。
   - 禁用按钮仍保留边框，避免视觉消失。

4. **禁止使用的颜色**：
   - 高饱和荧光色。
   - 与水墨风格冲突的渐变色（如彩虹渐变）。
   - 未在上述变量表中定义的颜色。

## 3. 字体规范

### 3.1 字体族

| 用途 | 字体 | fallback |
|------|------|----------|
| 标题 / 印章 / 叙事文案 | 宋体 / 书法体 | `"STSong", "SimSun", "Noto Serif SC", serif` |
| 正文 / UI 标签 / 按钮 | 系统无衬线 | `"PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif` |

### 3.2 字号层级

| 层级 | 字号（桌面） | 字号（移动端 ≤480px） | 用途 |
|------|--------------|----------------------|------|
| H1 主标题 | 56px | 40px | 标题页「都江堰治水」 |
| H2 副标题 | 20px | 18px | 标题页副文案、弹窗标题 |
| H3 小标题 | 18px | 16px | 关卡名、设置标题 |
| 正文 | 16px | 14px | 按钮文字、叙事文案 |
| 辅助说明 | 14px | 12px | 状态描述、占位说明 |
| 微标 | 12px | 12px | 金钱标签、尝试次数 |

### 3.3 排版规则

- 标题使用 `letter-spacing` 增加呼吸感（4px–12px）。
- 叙事文案使用 `line-height: 1.8`，居中对齐。
- 印章文字使用 `transform: rotate(-6deg)` 至 `rotate(-12deg)`，模拟手工盖印效果。

## 4. 组件复用规范

以下组件均已在 `prototype/index.html` 中实现，后续 Cocos UI 预制体需保持一致的视觉与交互参数。

### 4.1 按钮（`.btn`）

**基础样式**：
- 内边距：`12px 28px`（小按钮 `8px 16px`）。
- 边框：`2px solid var(--ink-dark)`。
- 底色：`var(--paper)`；主按钮 `--ink-dark` 底色 + `--paper` 文字。
- 圆角：`4px 12px 12px 4px`（卷轴式不对称圆角）。
- 投影：`3px 3px 0 var(--shadow)`。

**交互动效**：
- 悬停：`translateY(-2px)`，投影加深为 `4px 5px 0 var(--shadow)`，中心出现水墨晕开 radial-gradient。
- 按下：`translateY(0)`，投影恢复 `1px 1px 0 var(--shadow)`。

**状态**：
- 禁用：`opacity: 0.5`，`cursor: not-allowed`。

## 5. 页面布局规范

**视觉**：

- 主标题文字投影：`3px 3px 0 rgba(0,0,0,0.1)`。

### 5.3 LevelMapScene（关卡选择卷轴）

**结构**：
- 顶部 `.scene-header`：返回按钮 + 标题「治水卷轴」+ 占位宽度。
- 中部 `.scroll-map-container`：可横向滚动的卷轴区域。

**视觉**：

- 区域背景：中心 `--ochre` 径向渐变 + 纵向纸张渐变。

### 5.4 GameScene（游戏主界面）

**结构**（自上而下）：
1. `.game-header`：返回、关卡名、状态徽章、金钱条、撤销/重置。
2. `.game-canvas-area`：全屏 Canvas 编辑/模拟区。
3. `.game-toolbar`：构件卡片、删除区、放水按钮。

**Canvas 视觉层级**（从底到顶）：
1. 背景渐变（浅绿灰调，模拟河岸）。
2. 河流区域：横向矩形，`--water-light` 半透明渐变。
3. 河岸线：`--ink-dark` 0.4 透明度虚线。
4. 网格线（仅编辑阶段显示）：`--ink-dark` 0.08 透明度。
5. 水源标识、村庄触发区、支流引导虚线。
6. 构件（石墙 / 竹笼）。
7. 拖拽预览（半透明）。
8. 墨滴粒子。
9. 暂停遮罩（仅 Paused 状态）。

**村庄触发区**：
- 初始：`rgba(139, 58, 58, 0.15)` 填充 + 同色边框。
- 随命中比例增加：填充透明度升至 `0.7`，边框颜色加深。

## 6. 关键交互动效

所有动效参数均来自 `prototype/index.html`，后续实现必须保持一致。

### 6.1 场景切换

- 方式：直接切换 `.scene.active`（原型阶段无过渡动画）。
- 后续建议：Cocos 中可添加 `0.3s` 淡入淡出或横向滑入。
- **禁止**：使用夸张的 3D 翻转、缩放爆破等动效。

### 6.2 按钮反馈

- 悬停：上移 2px + 投影加深 + 中心水墨晕开。
- 按下：回弹 + 投影恢复。
- 时长：`0.2s ease`。

### 6.3 弹窗出现

- 当前实现：直接显示 `.modal-overlay.active`（无过渡）。
- 视觉：卷轴弹窗 + 黑色遮罩 `rgba(0,0,0,0.55)`。
- 后续建议：弹窗内容可从 `scale(0.95)` + `opacity: 0` 过渡到 `scale(1)` + `opacity: 1`，时长 `0.25s ease-out`。

### 6.4 关卡节点悬停

- 非锁定节点悬停：`translateY(-4px)`，`0.2s ease`。
- 锁定节点无悬停动效。

### 6.5 构件编辑动效

- 选中卡片：`translateY(-2px)`，背景变 `--green-pale`，投影加深。
- 放置构件：直接出现在网格位置（原型无缩放动画）。
- 点击旋转：立即切换 `rotStep`（90° 步进）。
- 拖拽移动：半透明跟随指针，释放后吸附网格。
- 拖入删除区：删除区 `scale(1.05)` + 背景加深。

### 6.6 水流与模拟动效

- 粒子生成：水源按 `base → peak → stable` 流量曲线生成墨滴。
- 粒子运动：水平速度 80–110 px/s，带轻微 y 轴随机偏移与重力下沉。
- 碰撞反馈：
  - 石墙：粒子 x 方向反弹，`vy` 随机偏移；石墙压力累积至倒塌。
  - 竹笼竖放：粒子 x 减速，y 方向被推向上下两侧。
  - 竹笼横放：粒子 x 反弹。
- 石墙倒塌：构件透明度降至 `0.35`（Collapsed 状态）。
- 村庄受击：触发区红色透明度随命中比例从 0.15 升至 0.7。

### 6.7 加载动效

- BootScene：进度条宽度线性增长，`0.1s` 步进。
- LoadingOverlay：水滴图标 `pulse 1.5s infinite`（缩放 1.0 → 1.05）。

### 6.8 Toast 动效

- 入场：从 `translateY(20px)` + `opacity: 0` 到 `translateY(0)` + `opacity: 1`。
- 时长：`0.3s ease`。
- 停留：2 秒后自动消失。

## 7. 错误 / 加载中 / 空数据状态的视觉表现

### 7.1 加载中状态

| 场景 | 元素 | 视觉表现 |
|------|------|----------|
| 启动加载 | BootScene | 居中标题 + 进度条 + 「正在引水入渠…」文案。进度条 `--paper-dark` 底色，`--ink-dark` 填充。 |
| 关卡切换 | LoadingOverlay | 卷轴小弹窗 + 脉冲水滴图标 + 「引水入渠中…」文案。 |
| 数据读取 | 设置/存档 | 使用本地存储，无网络请求加载态；若读取失败回退到默认值。 |

### 7.2 错误 / 非法操作状态

| 错误类型 | 触发条件 | 视觉反馈 |
|----------|----------|----------|
| 金钱不足 | 放置构件时余额不足 | Toast：「金钱不足」 |
| 库存不足 | 选择或点击已空库存 | 构件卡片 `.empty`（透明度 0.45）；Toast：「库存不足」 |
| 位置占用 | 新构件与已放置构件重叠 | Toast：「位置已被占用」 |
| 未放置构件 | 点击放水但画布为空 | Toast：「请至少放置一个构件」 |
| 删除确认 | 清除存档 | 浏览器原生 `confirm` 二次确认弹窗 |
| 关卡未解锁 | 点击锁定节点 | Toast：「关卡未解锁」 |
| L2 占位 | 点击 L2 节点 | Toast：「开发中，敬请期待」 |

**错误状态设计原则**：
- 不使用红色全屏错误页。
- 不使用模态阻塞弹窗（除清除存档二次确认外）。
- 错误信息通过 Toast 轻量提示，玩家可立即继续操作。

### 7.3 空数据状态

| 场景 | 空数据表现 | 视觉处理 |
|------|------------|----------|
| 无存档 | 首次启动 | 使用默认进度（L1 解锁、L2 未解锁），不显示空状态页。 |
| 无构件 | 重置后 | 画布清空，工具栏库存恢复，无需额外空状态提示。 |
| 无历史 | 撤销栈为空 | 撤销按钮禁用，`cursor: not-allowed`，`opacity: 0.5`。 |
| 无后续关卡 | L2 之后 | L2 节点显示「开发中，敬请期待」，不渲染更多节点。 |

| 卷轴地图 | 横向滚动 | 保持横向滚动，节点间距不变 |
| 按钮点击区域 | 默认 | 保持 `44px` 以上触控区域 |

### 8.3 触控规范

- 禁用长按选中：`-webkit-tap-highlight-color: transparent`。
- 禁用文本选择：`user-select: none`。
- Canvas 使用 Pointer Events 统一处理鼠标与触摸。
