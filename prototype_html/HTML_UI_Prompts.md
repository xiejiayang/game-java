## 1. 角色与目标

你是一个前端原型专家与 UI 设计师。请基于下方的都江堰治水题材解谜小游戏 PRD，生成一套**完整的 HTML 可交互原型文件**（单页或多页均可，优先单页 SPA 形式），用于：

1. 验证核心玩法循环：编辑 → 放水 → 模拟 → 结算。
2. 验证关键 UI 布局：标题页、关卡选择卷轴、游戏主界面、叙事弹窗、结算弹窗、设置弹窗、提示弹窗。
3. 为后续 Cocos Creator 3.x 2D 项目提供视觉与交互参考。

**生成要求**：
- 严格遵循 PRD 中的页面清单、流程、状态机、构件系统、水流模拟规则。
- 不得杜撰 PRD 中未出现的机制、页面、数值或美术资源。
- 占位内容需明确标注为「占位」，并在代码中以注释说明。
- 优先使用原生 HTML + CSS + JavaScript（可引入 Tailwind CSS 或轻量 UI 库加速样式，但不允许引入重型框架）。
- 原型需可在主流浏览器中直接打开运行。

## 2. 产品背景（不可修改的核心设定）

### 2.1 一句话描述
一款面向主流浏览器与微信小游戏的 **国风水利解谜小品**，让玩家在 60–90 秒的短关卡中通过“摆放治水构件 → 放水观察 → 失败/顿悟 → 再试”的循环，主动领悟李冰“堵不如疏、顺势而为”的治水智慧。

### 2.2 风格与平台
- **风格**：水墨国风 + 2D 横板表现。
- **平台**：第一阶段为主流浏览器 H5；第二阶段为微信小游戏。
- **付费原则**：不卖资源、不卖跳关、不付费跳过；仅允许激励视频广告换提示，以及外观/主题付费。
- **核心体验**：可顿悟的确定性解谜，而非硬核物理模拟。

### 2.3 目标用户
- 年龄 12–35 岁，休闲玩家。
- 单局时长 60–90 秒。
- 期望在碎片化时间获得「有意义的好玩」体验。

## 3. 原型范围（严格按 MVP）

### 3.1 包含内容

| 模块 | MVP 原型内容 |
|------|--------------|
| **关卡** | L1 完整可玩原型；L2 仅作为占位节点显示「开发中，敬请期待」。 |
| **核心解谜** | 编辑 → 放水 → 模拟 → 结算的完整流程。 |
| **构件系统** | 石墙、竹笼。石墙体现“硬堵失败”，竹笼体现“分水疏导”。 |
| **水流模拟** | 2D 简化粒子系统，视觉用墨滴粒子；结算判定与粒子表现解耦。 |
| **金钱系统** | 金钱上限、放置扣钱、删除返还、节俭判定。 |
| **关卡选择器** | 卷轴/地图形式，显示 L1、L2 状态。 |
| **存档** | 使用 localStorage 模拟本地存档；微信小游戏字段预留。 |
| **叙事** | 每关进入、成功、失败各一句文案弹窗。 |
| **提示** | 单层提示：L1 失败 1–2 次后触发老河工提示。 |
| **音效/美术** | 占位级，使用 CSS/SVG/Canvas 近似水墨风格，不引入真实资源。 |

### 3.2 不包含内容（禁止生成）

- L3–L8 关卡。
- 完整碑廊系统（仅允许通关后一句成就文案）。
- 三层提示树、复杂剧情演出。
- 真 3D 运行时渲染（3D 预渲染为 2D 精灵图，原型阶段用占位图代替）。
- 多人排行榜、社交裂变（仅预留数据字段）。
- 激励视频广告的真实接入（仅允许按钮占位）。

## 4. 页面 / 场景清单与 UI 要求

### 4.1 S01 BootScene（启动页）

**元素**：
- 全屏水墨背景（CSS 渐变或 SVG 占位）。
- 居中 Logo：游戏标题「都江堰治水」（占位字体：思源宋体或系统衬线体）。
- 底部版本号：v0.1.0 MVP。
- 加载进度条（占位动画）。
- 文案：「正在引水入渠…」。

**交互**：
- 自动播放 2–3 秒后淡入标题页 S02。
- 提供「跳过」按钮（可选，用于开发测试）。

**禁止杜撰**：不要添加注册页、隐私协议弹窗、第三方登录按钮（MVP 浏览器版为游客模式）。

---

### 4.2 S02 TitleScene（标题页）

**元素**：
- 全屏水墨山水背景（占位，可使用 CSS 绘制远山轮廓）。
- 主标题：「都江堰治水」。
- 副标题：「堵不如疏，顺势而为」。
- 主按钮：「开始治水」。
- 次要按钮：「设置」。
- 角落：音效开关图标（占位）。

**交互**：
- 点击「开始治水」进入 S03 LevelMapScene。
- 点击「设置」打开 O04 SettingsOverlay。
- 按钮悬停有水墨晕开效果（CSS 过渡）。

---

### 4.3 S03 LevelMapScene（关卡选择卷轴）

**元素**：
- 卷轴式横向或纵向布局，背景为展开的宣纸纹理。
- 关卡节点：
  - **L1 节点**：显示「L1 · 堵」，可点击进入。未通关显示普通状态；通关后显示印章「悟」或「安」。
  - **L2 节点**：显示「L2 · 分」，默认锁定/灰显，文案为「开发中，敬请期待」。L1 通关后高亮但仍不可进入。
- 返回按钮：返回 S02。
- 顶部：玩家昵称（游客）与关卡总览占位。

**交互**：
- 点击 L1 进入 S04 GameScene，并弹出 O01 NarrativeOverlay（开场）。
- 点击 L2 toast 提示「开发中，敬请期待」。
- 卷轴支持鼠标/触摸滑动浏览。

---

### 4.4 S04 GameScene（核心游戏界面）

**整体布局（2D 横板）**：
- **背景层**：河道地形、远山、村庄位置。河道从左向右流，村庄在右下侧。
- **编辑区**：占据屏幕中部，显示 0.5 米网格（可开关显示）。
- **顶部状态栏**：
  - 左侧：返回按钮、关卡名「L1 · 堵」。
  - 中部：金钱条（当前余额 / 上限）。
  - 右侧：撤销、重置按钮。
- **底部工具栏**：可用构件及剩余数量。
  - 石墙图标 + 数量。
  - 竹笼图标 + 数量。
  - 删除区（垃圾桶图标）。
- **右下角**：「放水」按钮（圆形或卷轴按钮）。

**构件表现**：
- **石墙**：矩形水墨石块，可拖拽、可旋转 90°。
- **竹笼**：圆柱/矩形竹编构件，可拖拽、可旋转 90°；竖放时分流上下，横放时阻挡或改变 x 方向。
- 拖拽时构件半透明吸附网格；释放后贴合网格。
- 已放置构件点击一次旋转 90°（rotStep = (rotStep + 1) % 4）。
- 拖拽已放置构件到删除区，移除并返还库存与金钱。

**编辑阶段 UI 状态**：
- 顶部显示「Editing」。
- 工具栏可交互。
- 「放水」按钮可用。
- 撤销最多 20 步（原型可简化为无限撤销，但需注明差异）。

**放水阶段**：
- 点击「放水」后校验布局与金钱。
- 若合法：顶部状态变为「Simulating」，锁定编辑，「放水」按钮置灰显示「模拟中…」。
- 若不合法：toast 提示具体原因（如「金钱不足」「未放置任何构件」）。

**模拟阶段视觉（占位级）**：
- 使用 Canvas 或 CSS 动画表现墨滴粒子从左向右流动。
- 粒子与石墙碰撞后反弹；与竖放竹笼碰撞后向上下分流。
- 石墙受压超过阈值后播放倒塌动画并变为半透明/消失。
- 村庄区域用触发区表示，粒子进入时区域变红。

**结算阶段**：
- 模拟结束自动弹出 O02 ResultOverlay。

**暂停/继续（可选）**：
- 模拟阶段点击暂停按钮进入 Paused 状态，显示遮罩与「继续」按钮。

---

### 4.5 O01 NarrativeOverlay（叙事弹窗）

**触发时机**：
- 进入 L1 时：开场白。
- L1 成功时：成功文案。
- L1 失败时：失败文案（可与 O02 合并或分别显示）。

**元素**：
- 半屏或全屏半透明黑色遮罩。
- 居中竖排或横排古风文案框（宣纸底纹）。
- 文案示例：
  - 开场：「水势汹汹，村庄危在旦夕。老夫观此河，堵不如疏……」
  - 成功：「水去村安，此法可暂保无虞。」
  - 失败：「墙倒村淹，须另寻他法。」
- 底部「继续」或「再试」按钮。

**交互**：
- 点击任意处或按钮关闭弹窗。
- 开场弹窗关闭后进入编辑阶段。

---

### 4.6 O02 ResultOverlay（结算弹窗）

**元素**：
- 居中弹窗，水墨卷轴边框。
- 标题：
  - 成功：「暂时安全」。
  - 失败：根据原因显示「村子仍被淹」或「墙倒了」。
- 信息行：
  - 实际消耗金钱：X / 上限。
  - 节俭判定：若 consumedMoney ≤ frugalMoney，显示「节俭治水」印章；否则显示「尚可」。
  - 尝试次数：attempts。
- 按钮组：
  - 「重试」：返回编辑阶段。
  - 「返回关卡」：回到 S03。
- 通关成功时额外显示知识点命名（如「分水导流」）作为最简碑廊占位。

**交互**：
- 点击「重试」关闭弹窗，重置关卡到编辑状态。
- 点击「返回关卡」跳转 S03。
- L1 通关后更新 S03 中 L2 节点状态（高亮但仍不可进入）。

---

### 4.7 O03 HintOverlay（老河工提示）

**触发时机**：
- L1 失败 1–2 次后，下一次进入编辑阶段或失败结算时触发。

**元素**：
- 半屏对话框，左侧老河工头像占位（水墨简笔人物剪影）。
- 提示文案：
  - 「巨石横亘，水压必溃。试试竹笼竖放，引水分流。」
- 高亮提示：用箭头或光圈高亮工具栏中的竹笼、河道关键区域。
- 「知道了」按钮。

**交互**：
- 点击关闭，返回编辑阶段。
- 提示次数 +1，本地记录避免重复频繁触发。

---

### 4.8 O04 SettingsOverlay（设置弹窗）

**元素**：
- 卷轴弹窗。
- 开关项：
  - 音效：开 / 关（占位）。
  - 音乐：开 / 关（占位）。
  - 语言：简体中文（占位，仅一种）。
- 按钮：
  - 清除存档（需二次确认）。
  - 关闭。

**交互**：
- 清除存档后重置 localStorage 中的关卡进度。
- 关闭后返回调用页面。

---

### 4.9 O05 LoadingOverlay（加载遮罩）

**元素**：
- 全屏半透明遮罩。
- 水墨加载动画（旋转的墨滴或波浪）。
- 文案：「引水入渠中…」。

**交互**：
- 场景切换时显示，切换完成后自动隐藏。

## 5. 核心流程（原型必须实现）

```text
启动(Boot) → 标题页(Title) → 关卡选择器(LevelMap) → 进入 L1 → 叙事开场(Narrative) → 编辑阶段(Editing) → 放水 → 模拟阶段(Simulating) → 结算(Settling) → 重试/返回
```

### 5.1 状态转换图（必须遵守）

```text
Editing ──[点击放水/布局合法]──> Simulating
Simulating ──[时间到/触发条件]──> Settling
Settling ──[点击重试]──> Editing
Settling ──[点击返回]──> LevelMapScene
Simulating ──[点击暂停]──> Paused
Paused ──[点击继续]──> Simulating
```

### 5.2 L1 关卡关键机制（不可偏离）

- **地形**：河道从左向右流，村庄在右下侧。
- **库存**：若干石墙、若干竹笼（具体数量占位，但需可配置）。
- **教学目标**：玩家硬堵失败后领悟「堵不如疏」。
- **石墙倒塌**：石墙受到水压超过 collapseThreshold 后，0.8–2 秒内倒塌；村庄约 10 秒内被淹。
- **成功路径**：用竹笼竖放分水，将主流导向远离村庄的支流。
- **节俭阈值**：实际消耗金钱 ≤ frugalMoney。

### 5.3 水流模拟（原型简化但需保留概念）

- 使用 Canvas 2D 或 DOM 粒子表现墨滴。
- 水源按 base → peak → stable 流量曲线生成粒子。
- 粒子沿 x 轴从左向右运动。
- 碰撞规则：
  - 石墙：x 方向反弹并衰减，对墙产生水势压力。
  - 竹笼：竖放时分流向上下（y 方向）；横放时阻挡或改变 x 方向。
- 石墙水势 ≥ collapseThreshold 后进入 Collapsed 状态，不再阻挡水流。
- 村庄触发区统计进入粒子数，达到失败阈值后判定失败。
- 最大模拟时长结束后进入结算。

**原型简化声明**：
- 无需严格物理引擎，粒子和判定可用简化逻辑。
- 必须保持判定与渲染解耦：结算逻辑不依赖粒子数量。

## 6. 美术与音效占位规范

### 6.1 美术风格

- **整体**：水墨国风、宣纸底纹、墨色渐变、留白。
- **色彩主调**：黑、白、灰、淡赭石、淡青绿。
- **字体**：标题用衬线体或书法体（系统字体 fallback）；正文用无衬线体保证可读性。
- **按钮**：卷轴边角、水墨晕开悬停效果、印章式图标。
- **粒子**：墨滴为黑色/深蓝灰色圆点，带透明度变化。
- **构件**：
  - 石墙：灰黑色矩形块，带纹理线。
  - 竹笼：黄褐色矩形或圆柱形，带竹编交叉线。

### 6.2 音效占位

- 在代码中用注释标注音效触发点，不引入真实音频文件。
- 触发点包括：放置构件、删除构件、旋转构件、放水、成功、失败、墙倒塌、按钮点击。

## 7. 技术要求

### 7.1 文件结构建议

```text
prototype/
├── index.html          # 入口，包含所有页面容器
├── css/
│   └── style.css       # 水墨国风样式
├── js/
│   ├── app.js          # 场景切换与状态管理
│   ├── levels.js       # 关卡配置（L1 配置、L2 占位）

    └── placeholder.svg # 占位图片（可选）
```

### 7.2 代码约束

- 使用原生 JavaScript，不引入 React/Vue/Angular。
- 可引入 Tailwind CSS CDN 或纯 CSS，但水墨效果优先用自定义 CSS。
- 必须响应式，适配移动端浏览器竖屏与横屏。
- 使用 Canvas 或 DOM 实现水流模拟均可，需保证 30fps 以上。
- localStorage 键名：
  - `dujiangyan_progress`：关卡进度与尝试次数。
  - `dujiangyan_settings`：音效、音乐开关。

## 8. 可直接使用的生成提示词模板

### 模板 A：生成完整可交互 HTML 原型

```text
请基于以下 PRD 生成一个完整的 HTML 可交互原型（单页 SPA，原生 JS + CSS）：

【产品】都江堰治水题材国风解谜小品，MVP 仅包含 L1 可玩 + L2 占位。
【核心循环】编辑构件 → 放水 → 模拟水流 → 结算。
【风格】水墨国风、2D 横板、宣纸底纹、墨色粒子。
【必须包含的页面】BootScene、TitleScene、LevelMapScene、GameScene，以及 NarrativeOverlay、ResultOverlay、HintOverlay、SettingsOverlay、LoadingOverlay。
【核心机制】
- 构件：石墙（硬堵，会倒塌）、竹笼（竖放分水、横放阻挡）。
- 操作：拖拽放置、点击旋转 90°、拖拽到删除区移除。
- 金钱：放置扣钱、删除返还、节俭判定。
- 水流：Canvas 2D 墨滴粒子，从左向右流，碰撞后反弹或分流；石墙受压倒塌；村庄触发区被淹则失败。
- 提示：L1 失败 1–2 次后触发老河工提示。
【禁止】不要生成 L3+ 关卡、不要真实广告接入、不要第三方登录、不要完整碑廊、不要 3D 渲染。

要求：
```
